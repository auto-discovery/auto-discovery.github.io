Methods experiments

Two selection methods built from the analysis findings, run head-to-head against PUCT-knob ablations on all three tasks. Two hybrids implemented but pending compute.

Method 1 — Rep-novelty PUCT bonus

Mechanism

score(i) = Q(i) + puct_c · scale · P(i) · √(1+T)/(1+n[i])      # existing PUCT
                + puct_c_rep · scale · z(novelty(i))            # ← new additive term

novelty(i):

  1. Frozen encoder: rollout response → Qwen3-8B with LoRA disabled (so the encoder doesn't drift during training) → mean-pool last-layer hidden state → 3584-d
  2. Compress to 64-d via sparse random projection (Li et al. 2006, density 1/√d_in, no fitting needed)
  3. Maintain online elliptical covariance: A = λI + Σ γ^(T-t) hₜhₜᵀ with γ=0.97 decay
  4. Score: novelty(i) = √(hᵀ A⁻¹ h) — LinUCB / OFUL confidence width
  5. Robust-standardize across candidates (median/MAD) → nov_z

Tracker observes every picked state, so the next selection's novelties are computed against past picks. Code: ttt_discover/methods/rep_novelty.py.

Why it should work

  • D6: reward ⊥ novelty (Kendall τ ≈ 0) — bonus injects independent signal
  • H3: scaled by scale (the candidate value range) so it survives standardization against Q
  • H4b/d: novelty rewards picks in directions A⁻¹ poorly covers — actively pushes the centroid into new regions

Method 2 — CVT-MAP-Elites QD archive

Mechanism

Replaces PUCT's flat buffer + ancestry-based selection with a behavior-cell archive:

  1. Warmup: run PUCT for ~5-7 batches until 80 valid (correctness>0) rollouts have descriptors
  2. Fit CVT once: k-means on the 80 descriptors → 64 centroids in 64-d rep space. Centroids frozen for rest of training.
  3. Each cell keeps one elite: the highest-reward state whose descriptor falls in that cell
  4. Selection: pick a cell via softmax of z(quality)/tau + c_rep·z(cell_novelty). Cell-novelty reuses the same LinUCB tracker but observes cell centroids of picked parents.
  5. The cell's elite becomes the next parent for rollouts

Config: n_cells=64, tau=0.5, c_rep=1.0. Code: ttt_discover/methods/qd_archive.py.

Mechanism-level analysis

The methods affected the diagnostic signals in interesting ways:

Interpretation

  1. The diagnostic was correct. PUCT's structural exploration collapses (H3); search becomes locally entropic (H4a) but globally stuck (H4b-d); reward ⊥ novelty (D6). All four motivate adding a rep-novelty signal.
  2. Rep-novelty PUCT bonus works on AC1 (highest headroom among "doable" tasks) — best result of all variants by a clean margin.
  3. QD-archive wins erdos by reframing exploration as cell coverage. Win is small (0.000017 over the next-best variant) and doesn't generalize.
  4. AC2 is the broken task — saturates 0.034 below frontier on every method tried. Active unlearning of useful directions; needs an objective-side intervention, not a selection-side one.
  5. No method beat the published 0.380876 frontier on erdos — closest was qd-archive at 0.380966 (+0.000090 short).

The caveat to all of this

Batch-size dependency. Every "method" run above was at 64 rollouts/step (small batch). Phase A is running the baselines at 512 rollouts/step (paper-matched).
  • erdos baseline at 512 rollouts already hits 0.380965 — better than every prior method, with no method enabled.
  • This suggests the "method wins" were measuring a weak baseline, not a strong method. The H1-H4 mechanisms are still real; they just cost less at proper batch.
  • Phase A is currently running 5 seeds × 3 tasks at full batch + 50 epochs — when it finishes we'll have the actual noise floor for honest method comparison.
